 Implementation Plan: Agent Evaluation Harness

 Context

 We're building a research-grade agent evaluation harness for aviation safety, drawing on patterns from the Blazer project (~/PycharmProjects/blazer). The repo
 is currently a clean slate — only .venv/, .idea/, plans/, and tasks/ exist. Python 3.10.11, Node.js 25.3, npm 11.7, and uv 0.9.22 are available.

 The full PRD is at tasks/prd-agent-eval-harness.md and the detailed task list (66 sub-tasks) is at tasks/tasks-agent-eval-harness.md. Rather than restating
 everything, this plan focuses on execution order, key decisions, and implementation patterns.

 Key Decisions

 1. Package manager: uv (already installed) for Python deps
 2. Promptfoo provider interface: call_api(prompt, options, context) -> dict returning {output, metadata, tokenUsage} — matches blazer's pattern exactly
 3. Config generation: YAML via yaml.safe_dump(), same as blazer
 4. Project layout: Flat (not nested under agent-evals/ subdir) — the repo root IS the project
 5. CLI framework: Click
 6. Aviation domain constants: Centralized in evals/aviation_domain.py

 Execution Plan

 We'll implement in 6 batches, following the task list dependency order. Each batch groups tasks that can be worked on together.
